**Dining Philosophers Problem**

*The Dining Philosophers Problem is a classical concurrency problem in computer science introduced by Edsger W. Dijkstra in 1965. It models the challenges of resource sharing among multiple processes that require exclusive access to limited resources, illustrating the concepts of deadlock, starvation, and synchronization.*


***Key facts***

Introduced by: Edsger W. Dijkstra (1965)

Core issues: Deadlock, starvation, mutual exclusion

Typical model: 5 philosophers, 5 shared chopsticks

Common solutions: Semaphores, monitors, resource hierarchy, or “waiter” algorithm
